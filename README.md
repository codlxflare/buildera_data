# MacroData — ИИ-ассистент по данным застройщика

ИИ-ассистент, который понимает **любые формулировки** на естественном языке и возвращает **корректно структурированные** ответы по данным MacroData (заявки, сделки, объекты, финансы, звонки, планы продаж и т.д.).

**Подключение к BI (Power BI и др.) для работы ассистента не требуется** — ассистент обращается к данным напрямую (MySQL MacroData или API).

## Запуск

```bash
# Установка зависимостей
npm install

# Ключ OpenAI — в .env.local (файл в .gitignore, не коммитить).
# Если файла нет: скопируйте .env.example в .env.local и укажите OPENAI_API_KEY.
# Опционально: OPENAI_CHAT_MODEL (по умолчанию gpt-4o для лучшего понимания контекста и точности).
# Если ключ когда-либо попадал в чат или репозиторий — смените его в личном кабинете OpenAI.

# Режим разработки
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000) — откроется чат-интерфейс. Задайте вопрос по заявкам, сделкам, объектам и т.д. Пока база MacroData не подключена, ассистент объясняет, какие данные будут доступны после подключения.

**Продакшен:** `npm run build` и `npm start`.

## Что уже есть

- **Чат-интерфейс** — тёмная тема, подсказки, отправка по Enter.
- **API `/api/chat`** — приём вопроса, вызов OpenAI с контекстом схемы MacroData, ответ на русском.
- **macrodata_schema_reference.md** — справочник схемы MacroData v1.1.73 для маппинга вопросов на таблицы и поля.

## Доступ к данным (для ассистента)

- **MySQL** (выделенная БД MacroData): хост, порт, пользователь, пароль, имя БД — добавить в `.env.local` когда реквизиты будут (см. `.env.example`).
- **API** — при наличии: эндпоинты, авторизация, формат ответов.

### Схема и примеры данных (без изменений в БД)

- **api.txt** — описание схемы MacroData (таблицы, поля, связи). Используется для генерации SQL: ИИ знает список таблиц и полей. Файл не меняется кодом; при обновлении схемы БД обновите api.txt вручную.
- **api_samples.txt** — примеры значений из БД (только чтение). Генерируется автоматически при первом запросе к чату с подключённой БД или вручную: `GET /api/samples?refresh=1` (выгрузка в файл и в ответ). По примерам ИИ видит реальные статусы (например finances.status_name «к оплате»), форматы дат, названия каналов — меньше ошибок в запросах. БД при выгрузке не изменяется.

## Дальнейшие шаги

1. Получить реквизиты доступа к MacroData (MySQL и/или описание API).
2. Реализовать слой доступа к данным (MySQL и/или API).
3. Реализовать логику ассистента: **любая фраза пользователя** → интерпретация намерения → выбор таблиц/полей/агрегаций → запрос к данным → **структурированный ответ** (таблица, сводка, метрики).

## Примеры вопросов, которые должен поддерживать ассистент

- Сколько заявок за последний месяц по источникам?
- Сумма и количество проведённых сделок за квартал по отделам?
- Какие объекты в продаже в ЖК X по цене до Y?
- Сколько броней и отмен за период?
- Поступления по сделкам за месяц / в разрезе домов?
- Конверсия из заявки в сделку по менеджерам?
- План vs факт по продажам по домам/менеджерам?
- Расходы на рекламу по каналам и UTM?

Схема в `macrodata_schema_reference.md` уже ориентирована на такие запросы.
